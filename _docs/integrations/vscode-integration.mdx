# Introduction
EkLine highlights violations to your company style guide with Visual Studio Code diagnostics. It provides real-time feedback, helping you maintain consistent writing throughout your documentation.

The EkLine VSCode Extension uses EkLine to highlights violations as you make changes.

# Prerequisites

You need an EkLine.io API token to use this extension.

# Getting Started

## Installation

To install the vscode extension from web, follow these steps:

1. Navigate to the [EkLine Extension Page](https://marketplace.visualstudio.com/items?itemName=EkLine.ekline-vscode) in the VSCode Marketplace:
https://marketplace.visualstudio.com/items?itemName=EkLine.ekline-vscode
2. Click Install.
3. Open Visual Studio Code when prompted.

OR,

To install the vscode extension from within vscode,

1. Open Visual Studio Code.
2. Go to the Extensions view by clicking on the Extensions icon in the Activity Bar on the side of the window.
3. Search for "EkLine" in the Extensions view search box.
4. Click the Install button next to the extension with the same name.

## Configure EkLine linter

On first activation, EkLine prompts for an API Token.  


If cancelled or the token is invalid, the extension displays a popup providing additional opportunity to configure EkLine or disable to prevent future popups.
When disabled, run 'Configure EkLine Linter' from the Command Pallete or clear the 'disabled' box in Extension Settings. 


# Usage 

The extension runs EkLine on supported files when openend or saved.  

You can analyze all supported files in the workspace by invoking 'Run EkLine Linter' from the Command Pallette
